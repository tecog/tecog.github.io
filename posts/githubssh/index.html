Github设置SSH key后推送依然要求输入用户名和密码

2022.12.21

推送代码到Github时发现要求登录，明明已经设置过了SSH，而且另外的项目也可以正常推送，故判断是项目配置问题，一番调查后找到原因

因为拉取代码用的是https而不是ssh，故需要更新origin为ssh格式 HTTPS的格式为

https://github.com/用户名/仓库名.git

SSH的格式为

git@github.com:用户名/仓库名.git

修改方法

git remote remove origin
git remote add origin git@github.com:用户名/仓库名.git

再次推送时就会采用SSH的方式推送代码。 可以查看config文件的[remote "origin"]验证
